# Feed Video Autoplay

스크롤 시 피드 내 비디오를 자동으로 미리보기(5초) 재생하고, 성능을 극대화하기 위한 프론트엔드 최적화 구현체입니다.

## 🚀 주요 구현 방식

- 스크롤을 감지하여 화면 중앙(약 40% 지점)에 가장 가까운 비디오 카드를 찾습니다.
- 해당 비디오 카드만 5초간 자동 재생(미리보기)하며, 이후에는 정지하고 썸네일로 복귀합니다.
- 다수의 카드(가로형, 세로형, 그리드형 등)가 화면에 있을 때 재생 우선순위(위에서 아래, 왼쪽에서 오른쪽)를 계산하여 순차적으로 재생합니다.

## 🛡️ 프론트엔드 최적화 내용 (메모리 방어 및 성능 향상)

풀 영상을 제공받아 클라이언트에서 5초만 자르는 환경에서 발생하는 네트워크 낭비와 메모리 부하를 방어하기 위해 다음과 같은 기법들이 적용되었습니다.

### 1. 지연 로딩 (Lazy Loading)
- 비디오 태그에 `preload="none"`을 설정하고, 실제 소스 URL은 `data-src`에 보관합니다.
- 화면에 진입하여 재생이 필요해질 때 비로소 `src` 속성에 URL을 할당하여 불필요한 초기 다운로드를 완벽히 차단합니다.

### 2. 미디어 프래그먼트 (네트워크 버퍼링 방어)
- 재생 시 URL 끝에 `#t=0,5` 파라미터를 추가하여 브라우저에게 "처음 5초까지만 필요하다"는 힌트를 제공합니다.
- 모바일 환경 등에서 전체 영상을 백그라운드로 버퍼링하는 현상을 막아 네트워크 데이터 낭비를 대폭 줄입니다.
- 사용자가 '전체 보기'를 클릭할 경우 프래그먼트를 제거하고 원본 URL로 교체해 끝까지 재생합니다.

### 3. 초강력 메모리 해제 (Intersection Observer)
- 스크롤 이벤트에만 의존하지 않고, `IntersectionObserver`를 도입해 카드가 뷰포트 위아래 500px 밖으로 벗어나는 즉시 자원을 해제합니다.
- 단순히 `pause()`만 하는 것이 아니라 `video.removeAttribute('src')`와 `video.load()`를 호출해 진행 중이던 스트리밍(네트워크 캐시) 연결을 강제 종료하고 브라우저 메모리를 완전히 비워버립니다.

### 4. DOMException 방어
- 사용자가 스크롤을 아주 빠르게 넘겨 영상이 `play()` 되자마자 `pause()`가 호출될 때 발생하는 `AbortError`를 정상적인 흐름으로 간주하여 `catch` 구문에서 무시(방어)합니다. 쓸데없는 콘솔 에러와 실행 지연을 막습니다.

### 5. GPU 가속 (하드웨어 렌더링)
- CSS에 `transform: translateZ(0);` 및 `will-change: opacity;`를 적용하여 비디오 엘리먼트 렌더링을 CPU가 아닌 GPU로 강제 이관했습니다. 
- 복잡한 렌더링 환경과 용량이 큰 미디어 뷰에서도 스크롤 버벅임(Jank) 없이 부드러운 스크롤을 유지합니다.
